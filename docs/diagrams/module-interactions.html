ЛИСТ 02

Один флэш: Flash.calculate()

Что передаётся между модулями внутри одного вызова — от состава и условий до FlashResult.

Composition(zi, T_res, eos_name) + Conditions(p_bar, t_C)
вход: мольные доли, давление, температура
↓composition, conditions
Flash.calculate()
calc_core/VLE/Flash.py
↓composition, P, T
TwoPhaseStabilityTest.calculate_phase_stability()
внутри — BrusilovskiyEOS считает фугитивности пробных фаз (Wilson K-старт)
↓stable: bool, k_values, S_v, S_l
если unstable → 2 фазы
PhaseEquilibriumNewton.find_solve_loop()
Rachford-Rice (Fv) → Ньютон по ln(fugacity)
↓xi_l, yi_v, Fv
FluidPropertiesCalculator ×2
по жидкости и по пару отдельно
↓M, Vm, ρ, Z, μ (LBC)
FlashResult(vapor, liquid, is_two_phase=True)
если stable → 1 фаза («ТРЮК»)
весь состав → жидкость
vapor.mole_fraction = 0.0, без проверки газ/жидкость
↓composition.composition
FluidPropertiesCalculator ×1
↓M, Vm, ρ, Z, μ
FlashResult(is_two_phase=False)
PhaseIdentificator мог бы уточнить фазу, но не интегрирован в Flash
ЛИСТ 03

Эксперимент по ступеням

DLE / CCE / SeparatorTest — общий скелет, состав жидкости переходит со ступени на ступень.

Composition (пластовый) + список ступеней
давления (DLE/CCE) или пары P/T (SeparatorTest)
↓composition, T
SaturationPressure.sp_convergence_loop()
PhaseDiagram/new_methodv2.py — напрямую через TwoPhaseStabilityTest, минуя Flash
↓P_sat
цикл по ступеням i = 1…N
Flash(composition_i, conditions_i).calculate() → FlashResult_i
↻composition.new_composition(xi_l)
жидкость со ступени i → состав ступени i+1
пар со ступени считается «ушедшим», не участвует дальше
↓[FlashResult_1 … FlashResult_N]
_vectorize_dle_results(...)
Bo, Rs и т.п. — код дословно продублирован между DLE.py и SeparatorTest.py
ЛИСТ 04

Зона дублирования: PhaseDiagram/

Единственная область с 4–5 параллельными реализациями одной задачи. Ни одна не канонична.

используется в проде new_methodv2.py SaturationPressure

Вызывается напрямую из DLE / CCE / SeparatorTest — де-факто основная версия.

независимая реализация BubblePointPressure.py + DewPressure.py BubblePointCalculator / DewPointCalculator

Используется в test_notebook.ipynb, но не в экспериментах.

версия Дениса SaturationPressure_den_v.py DewPointCalculator

В коде открытая пометка: «непонятно, откуда брать K-факторы».

используется в проде CriticalPoint.py CriticalPointCalculator

Критерий: минимизация зазора между bubble- и dew-кривой.

осиротевший CriticalProperties_den_v.py CriticalPointCalculator

Тот же класс, другой критерий (касательная плоскость). Единственный потребитель — сломанный CVD_den_v.py.

только в ноутбуке test_notebook.ipynb, ячейки «From VBA» / «Parallel» CriticalPointCalculator / PhaseDiagramCalculator

Ещё 2 реализации существуют только внутри ячеек, не вынесены в calc_core/.

При любой задаче, связанной с давлением насыщения, конденсации, критической точкой или фазовой огибающей — уточнять у пользователя, с какой реализацией работать, не выбирать самостоятельно.
