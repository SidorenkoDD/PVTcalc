# Бэклог развития PVTcalc

> Актуальная очередь будущих работ, оценки масштаба, зависимости и критерии
> готовности перенесены в [`ROADMAP.md`](ROADMAP.md). Этот файл сохраняется как
> подробная история ранее сформулированных и уже выполненных задач.

**Порядок работы с 2026-07-20:** задачи выполняются последовательно по
`ROADMAP.md`; после каждой проверенной итерации отметка обновляется в обоих
файлах. Автор утвердил KRSNLN будущим PVTSim-эталоном, сохранение результатов в
единой общей базе проектов и заморозил CVD, лампинг, EOS-regression и mixing-rule
до отдельного запроса. Текущие инженерные допуски Flash/Psat не менять.

Список намеченной работы по проекту, сгруппированный по темам. В отличие от `CLAUDE.md` (описывает **текущее** состояние репозитория — что живое, что легаси, какие баги известны), этот файл описывает **будущую** работу. При завершении задачи — вычёркивать чекбокс, а не удалять пункт (история решений полезнее пустого списка).

Приоритеты:
- **P0** — блокирует дальнейшее развитие (в первую очередь — построение интерфейса поверх движка) или является дешёвой мерой, которая окупается сразу.
- **P1** — важно, но не блокирует, можно параллелить с P0.
- **P2** — полезно, но не срочно; часть пунктов ждёт отдельного решения автора, прежде чем их вообще можно начинать.

Составлено 2026-07-15 по итогам анализа архитектуры, кодовой базы, тестирования и защитных механизмов (логирование/валидация). Актуальное состояние по каждому пункту сверяйте с кодом — бэклог фиксирует *задачи*, а не факты о репозитории (факты — в `CLAUDE.md`).

**Статус на 2026-07-19**: GUI-MVP стабилизирован без расширения предметной функциональности: dirty/save lifecycle, единый ModelStore, стабильная адресация фоновых задач, нормализованная сессия v3, валидация импортов, гранулярные StateChange-события и headless DPG-тесты. Исправлены согласованные известные баги ядра; весь набор — 145 тестов. Актуальные ограничения и структура перечислены в `GUI.md`, СТАТУС-11.

**Итерация R3.9 выполнена 2026-07-22**: Lab Data отделены от обязательного
владения моделью/запуском. Общие наборы уровня проекта видны отдельной веткой

дополнительно поддерживаются model-scoped наборы. Расчётный узел сохраняет
ссылку, а историческая локальная таблица остаётся read-only fallback. Наборы
создаются и правятся вручную прямо из
ветки Project/Model Lab Data с названием, условиями T/P_res и построчной
таблицей. В каждом scope дерево сгруппировано списками DLE/CCE/Separator;
переименование открывает редактор, удаление вызывается из контекстного меню или
клавишей Del в редакторе и требует подтверждения. Пустой либо частично
заполненный черновик не создаётся; первая полная точка и последующие правки
сохраняются автоматически. Имена новых наборов нумеруются (`DLE LAB-1` и далее);
одиночный клик выбирает набор, двойной открывает редактор. Расчёт
только выбирает готовый источник, а legacy-таблица показывается read-only.
Все текстовые input-поля имеют контрастную заливку; состояние редактора
очищается при любом закрытии. Системный крестик viewport заблокирован, поскольку
DearPyGui сообщает о нём слишком поздно для диалога; Project → Exit application
всегда даёт контролируемое подтверждение (Save/Discard/Cancel либо Close/Cancel).
Редактор показывает десять строк; в набор попадают только полные точки. Ячейка
выбирается одиночным кликом, стрелками переносится выделение, двойной клик или
Enter активируют постоянное поле Value, следующий Enter фиксирует значение.
Таблица и действия используют
приглушённую палитру светлой темы.
Промежуточная публикация из расчёта исключена из UI. Файловый импорт не
разрабатывается, но выбранная ячейка принимает Excel-столбец или таблицу через
Paste from Excel/Ctrl+V без очистки соседних колонок. Новый GUI-каталог лежит в атомарном
`models.lab.json` рядом с `models.json`, не нарушая формат, используемый
расчётным ядром; графики и межмодельный Compare разрешают эти ссылки.
Полный набор после итерации — **294 теста зелёные**; `ruff` и `mypy gui`
проходят.

**ModelStore 2026-07-18 выполнен**: все живые пути `models.json` используют версионированную валидацию, атомарную запись, lock, optimistic concurrency и три ротационных backup. Битая база диагностируется в Projects без падения и без автоматической перезаписи. Полный набор — 139 тестов.

**Гранулярный рендер 2026-07-19 выполнен**: `AppState` публикует типизированный `StateChange`, а View обновляет только Projects, дерево, workspace или body конкретной вкладки по области события. Фоновый `NODE`-update сохраняет контейнеры соседних вкладок; восстановление модели больше не вызывает двойную пересборку. Полный набор — 141 тест.

**Восстановление сессии 2026-07-19 выполнено**: `SessionState` и `workspace_codec` санируют повреждённые структуры и восстанавливают валидные части без падения Projects/workspace. Полный набор — 145 тестов.
