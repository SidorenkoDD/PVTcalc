# Бэклог развития PVTcalc

Список намеченной работы по проекту, сгруппированный по темам. В отличие от `CLAUDE.md` (описывает **текущее** состояние репозитория — что живое, что легаси, какие баги известны), этот файл описывает **будущую** работу. При завершении задачи — вычёркивать чекбокс, а не удалять пункт (история решений полезнее пустого списка).

Приоритеты:
- **P0** — блокирует дальнейшее развитие (в первую очередь — построение интерфейса поверх движка) или является дешёвой мерой, которая окупается сразу.
- **P1** — важно, но не блокирует, можно параллелить с P0.
- **P2** — полезно, но не срочно; часть пунктов ждёт отдельного решения автора, прежде чем их вообще можно начинать.

Составлено 2026-07-15 по итогам анализа архитектуры, кодовой базы, тестирования и защитных механизмов (логирование/валидация). Актуальное состояние по каждому пункту сверяйте с кодом — бэклог фиксирует *задачи*, а не факты о репозитории (факты — в `CLAUDE.md`).

**Статус на 2026-07-18**: GUI-MVP стабилизирован без расширения предметной функциональности: dirty/save lifecycle, атомарные JSON-записи, стабильная адресация фоновых задач, сессия v3, валидация импортов и headless DPG-тест. Исправлены согласованные известные баги ядра; весь набор — 127 тестов. Актуальные ограничения и структура перечислены в `GUI.md`, СТАТУС-6.

Единицы температуры разведены (K vs °C) по фактическим сигнатурам методов. **Не покрыто**: внутренние мутации `new_composition()` внутри самих экспериментов (там доли всегда нормируются `_normalize_composition`, поэтому проверка избыточна) и `flash()`-внутренности.
- [x] **P0** — Явный сигнал о несходимости численных методов: решатели Ньютона в основном пайплайне должны явно бросать `ConvergenceError` вместо тихого возврата неверного результата. **Выполнено 2026-07-15**: `VLE/PhaseEquilibriumNewton.find_solve_loop()` теперь бросает `ConvergenceError` на исчерпании лимита итераций (`_FUG_MAX_ITER`, промотирован из хардкода 1000) вместо возврата несошедшегося dict; `PhaseStability/TwoPhaseStabilityTest` уже бросал `StopIterationError` (⊂ `ConvergenceError`); `PhaseEnvelopeFacade.bubble_point`/`dew_point`/`critical_point` уже поднимают `ConvergenceError`. Сознательно НЕ трогали: тривиальное решение (K→1 — штатный флаг, не мусор) и внутренний RR-решатель (несходимость проявится как несходимость внешнего цикла).
